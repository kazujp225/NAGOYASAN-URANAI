# CLAUDE.md — 占いプラットフォーム Web MVP 仕様書

## 0. このファイルの目的

このプロジェクトは、クライアントが構想している「占いに関するWebプラットフォーム」を、まずWebだけで最小限リリースできる形に落とし込むための仕様書である。

Claude Code / 開発AI は、この `CLAUDE.md` を最優先のプロジェクト指示として読み、実装・設計・リファクタリング・テストを進めること。

重要方針：

- MVPでは「Web完結」を最優先する。
- ネイティブアプリ開発は初期スコープ外。
- 手相の本格AI画像解析は初期スコープ外。MVPでは「撮影・アップロード体験」と「用意された占い結果表示」を優先する。
- 課金は初期から設計に入れるが、最初は単純な無料/有料プランに限定する。
- VTuber的な自動応答は、MVPでは「キャラクター付きチャットUI」まで。音声・Live2D・3Dモデルは後続フェーズ。
- 占い結果は医療・法律・投資・人生の重大決定を断定しない。娯楽/自己理解コンテンツとして表現する。

## 1. サービス概要

### 1.1 サービス名

仮称：`Fortune Platform`

正式名称は未定。コード上では一旦 `fortune-platform` として扱う。

### 1.2 コンセプト

スピリチュアル・占いに関心があるユーザー向けに、手相・名前・生年月日・居住地などをもとにした複数の占いを、ポップで親しみやすいUIで提供するWebサービス。

ユーザーは無料で簡易結果を確認でき、有料プランではより詳しい鑑定結果・日替わり占い・キャラクターによる深掘りコメントなどを閲覧できる。

### 1.3 ターゲットユーザー

主なターゲット：

- 占い、スピリチュアル、自己分析に興味がある人
- 恋愛・仕事・人間関係・金運などのテーマに関心がある人
- X/TikTok/Instagramで占いコンテンツを見て流入するライト層
- 「本格的すぎる占い」よりも、ポップで気軽な体験を求める人

避けるべき方向性：

- 不安を煽って高額課金に誘導する設計
- 「必ず当たる」「病気が治る」「投資で勝てる」などの断定表現
- 宗教・霊感商法に見える表現

## 2. MVPの範囲

### 2.1 MVPで作るもの

MVPでは以下を実装する。

1. LP / トップページ
2. 5種類の占いメニュー
3. 手相占いの撮影/アップロードUI
4. 名前・生年月日・居住地入力フォーム
5. 用意された結果データに基づく占い結果表示
6. 日替わり占い機能
7. 無料結果と有料結果の出し分け
8. 課金導線
9. 管理者用の簡易コンテンツ管理
10. キャラクター付き自動応答UIの簡易版
11. X拡散用の共有導線
12. 利用規約・プライバシーポリシー・免責表示

### 2.2 MVPで作らないもの

MVPでは以下を作らない。

- iOS/Androidネイティブアプリ
- App Store / Google Play 申請
- 本格的な手相AI画像解析モデル
- Live2D / 3D VTuber / 音声会話
- 複雑な占星術ロジック
- 高度なレコメンドエンジン
- 多言語対応
- LINE連携
- 高度なCRM/MA連携

### 2.3 なぜMVPを絞るか

初期から「手相AI」「ネイティブアプリ」「課金」「VTuber」「365日占い」「自動応答」を全部入れると、費用・期間・保守負荷が膨らむ。

最初は以下の検証を優先する。

- ユーザーが占い結果を最後まで読むか
- 手相撮影UIに抵抗がないか
- 有料詳細へのクリック率があるか
- Xから流入するか
- 毎日訪問する理由が作れるか

## 3. 占いメニュー仕様

MVPでは5種類の占いを提供する。

正式名称はクライアント確認後に変更可能。仮名称は以下。

### 3.1 手相占い

入力：

- 手のひら画像
- 利き手
- 年齢帯
- 性別任意
- 相談テーマ任意：恋愛 / 仕事 / 金運 / 人間関係 / 総合

MVP処理：

- 画像をアップロードまたはブラウザ上で撮影する。
- 画像そのものをAIで精密解析しない。
- 結果は、入力値・日付・ユーザーID・ランダムシードをもとに、事前登録された結果テンプレートから選択する。
- UI上では「手のひらを読み取っています」という演出を入れる。
- 結果表示時には、あくまでエンタメであることを明記する。

将来処理：

- 画像から生命線・感情線・頭脳線などを検出するAIモデルを導入。
- 撮影品質チェックを導入。
- 線の長さ・濃さ・枝分かれなどを特徴量化。

注意：

- MVPで「AIが手相を正確に解析している」と誤認させないこと。
- 画像保存の有無をユーザーに明示すること。
- 画像は初期では短期保存または保存しない設計を推奨する。

### 3.2 名前占い

入力：

- 名前
- ふりがな任意
- ニックネーム任意

出力：

- 性格傾向
- 恋愛傾向
- 仕事傾向
- 人間関係のクセ
- ラッキーアクション

MVP処理：

- 名前文字列をseed化し、テンプレートから結果を生成する。
- 姓名判断の厳密な画数計算はMVPでは必須ではない。
- 将来、画数ロジックを入れられるよう拡張余地を残す。

### 3.3 生年月日占い

入力：

- 生年月日
- 性別任意
- 相談テーマ任意

出力：

- 基本性格
- 今の運気
- 恋愛/仕事/金運の傾向
- 今日のアドバイス

MVP処理：

- 生年月日から day-of-year / 星座 / 数秘風の値を計算する。
- 結果テンプレートから該当結果を出す。

### 3.4 居住地・方位占い

入力：

- 都道府県
- 市区町村任意
- 引っ越し/旅行/仕事/恋愛などのテーマ任意

出力：

- 今の環境との相性
- おすすめ行動
- ラッキー方位風コメント
- 注意すべき生活リズム

MVP処理：

- 詳細な地理解析は行わない。
- 都道府県・地域カテゴリ・相談テーマからテンプレート結果を出す。

### 3.5 総合人生占い

入力：

- 名前
- 生年月日
- 居住地
- 手相画像任意
- 相談テーマ

出力：

- 総合運
- 恋愛運
- 仕事運
- 金運
- 人間関係
- 今月のテーマ
- 具体的なアクション3つ

MVP処理：

- 他4つの占い結果を統合したように見える文章を出す。
- 無料版では要約のみ。
- 有料版では詳細セクションを開放する。

## 4. 日替わり占い仕様

### 4.1 基本仕様

日替わり占いは、毎日変わる結果を表示する。

必要データ：

- 365日分の基本コンテンツ
- 各日のタイトル
- 総合運スコア
- 恋愛運スコア
- 仕事運スコア
- 金運スコア
- ラッキーカラー
- ラッキーアイテム
- 一言メッセージ
- 詳細メッセージ 有料用

### 4.2 結果の決定方法

未ログインユーザー：

- 日付ベースで共通結果を表示。

ログインユーザー：

- 日付 + ユーザーID + 生年月日などをseed化して、少しパーソナライズした結果を表示。

### 4.3 365日データが未完成の場合

開発中はダミーデータ可。

本番リリース前には以下が必須：

- 365件以上の `daily_fortunes` データ
- 無料本文と有料本文の両方
- 管理画面から修正可能な状態

## 5. 課金仕様

### 5.1 課金モデル

MVPではシンプルな2階層にする。

#### Free

- 5種類の占いの簡易結果
- 日替わり占いの一部閲覧
- 手相撮影UIの利用
- X共有

#### Premium

- 詳細結果の全文閲覧
- 総合人生占いの詳細表示
- 日替わり占いの詳細解説
- キャラクターによる追加コメント
- 過去の占い履歴閲覧

### 5.2 価格案

仮案：

- 月額：480円〜980円
- 単発詳細鑑定：300円〜1,500円
- 初期検証では月額よりも単発課金の方がCVRを確認しやすい可能性あり。

最終価格はクライアント確認事項。

### 5.3 MVP課金実装

推奨：

- Stripe Checkout または Stripe Payment Links を使用。
- Webhookで決済完了を受け取り、ユーザーの `subscription_status` または `entitlement` を更新する。
- 管理画面からユーザーの課金状態を確認できるようにする。

### 5.4 有料/無料の出し分け

占い結果ページは以下の構造にする。

- 無料部分：冒頭30〜40%を表示
- 有料部分：ぼかし/ロック表示
- CTA：「もっと詳しく読む」
- 課金後：同じページで全文表示

## 6. 自動応答 / キャラクター機能

### 6.1 MVP仕様

MVPではVTuber的な雰囲気を出すが、本格実装はしない。

実装内容：

- 画面右下または結果ページ上にキャラクターを表示
- 吹き出しでコメント
- ユーザーの占い結果に合わせて短いコメントを返す
- テキストチャット形式
- キャラクター画像は静止画または簡易アニメーション

### 6.2 応答内容

応答は以下に限定する。

- 占い結果の補足
- 今日の一言
- 結果の読み方
- 有料プランの案内
- サイト内ナビゲーション

禁止：

- 医療/法律/投資の助言
- 恐怖訴求
- 断定的な未来予測
- 依存を煽る表現

### 6.3 将来拡張

- Live2D
- 音声合成
- 音声入力
- 配信者風の待機モーション
- ユーザー履歴に基づいたパーソナライズ

## 8. UI/UX方針

### 8.1 デザイン方向性

キーワード：

- ポップ
- 神秘的だが重すぎない
- 夜空/星/手相/カード/オーラ風
- 女性向け寄りだが、過度に限定しない
- SNSで共有したくなる

### 8.2 避けるデザイン

- 怪しすぎる霊感商法風
- 黒と紫だけの重いホラー調
- 高齢者向け占いサイト風
- 情報商材LP風
- 過剰な不安訴求

### 8.3 コピー方針

良い例：

- 「今日のあなたに必要な一言」
- 「今の流れをやさしく読み解く」
- 「手のひらから、今のあなたの傾向をチェック」
- 「詳しい鑑定で、次の一歩を見つける」

悪い例：

- 「これを見ないと不幸になります」
- 「100%当たる」
- 「絶対に復縁できます」
- 「病気が治る」
- 「今すぐ課金しないと運気が下がる」

## 11. API設計

### 11.1 占い実行

`POST /api/readings`

request:

```json
{
  "fortuneType": "palm",
  "input": {
    "name": "",
    "birthdate": "1995-01-01",
    "prefecture": "Tokyo",
    "theme": "love",
    "dominantHand": "right"
  },
  "palmImageId": "optional"
}
```

response:

```json
{
  "readingId": "uuid",
  "result": {
    "title": "今のあなたは変化の入口にいます",
    "summary": "今日は人との会話からヒントが得られそうです。",
    "freeBody": "...",
    "premiumLocked": true,
    "scores": {
      "total": 78,
      "love": 72,
      "work": 81,
      "money": 64
    }
  }
}
```

### 11.2 手相画像アップロードURL発行

`POST /api/palm-images/upload-url`

request:

```json
{
  "fileName": "palm.jpg",
  "contentType": "image/jpeg",
  "consentAccepted": true
}
```

response:

```json
{
  "palmImageId": "uuid",
  "uploadUrl": "signed-url",
  "storagePath": "palm/user-id/uuid.jpg"
}
```

### 11.3 日替わり占い

`GET /api/daily-fortune?date=YYYY-MM-DD`

response:

```json
{
  "date": "2026-05-06",
  "fortune": {
    "title": "小さな選択が流れを変える日",
    "summary": "今日は迷ったら軽い方を選ぶと良い日です。",
    "premiumLocked": true
  }
}
```

### 11.4 キャラクター応答

`POST /api/chat`

request:

```json
{
  "readingId": "uuid",
  "message": "恋愛運をもっと詳しく教えて"
}
```

response:

```json
{
  "message": "今のあなたは、相手に合わせすぎないことが大切みたい。まずは自分の気持ちを言葉にしてみて。"
}
```

### 11.5 Stripe Webhook

`POST /api/stripe/webhook`

処理：

- checkout.session.completed
- customer.subscription.updated
- customer.subscription.deleted
- invoice.payment_failed

Webhookで `subscriptions` と `profiles.subscription_status` を更新する。

## 12. 占い結果生成ロジック

### 12.1 基本方針

MVPでは「複雑な占術エンジン」ではなく、「一貫性のあるテンプレート選択ロジック」を作る。

目的：

- 安く早くリリースする
- 表示結果の品質をコンテンツ側で担保する
- 後から本格ロジック/AIに差し替え可能にする

### 12.2 seed生成

入力値からseedを生成する。

例：

```ts
const seedSource = [fortuneType, name, birthdate, prefecture, theme, date].join(':')
const seed = hash(seedSource)
```

同じユーザーが同じ条件で実行した場合、毎回結果が大きく変わりすぎないようにする。

### 12.3 スコア生成

seedから以下を生成する。

- total: 1〜100
- love: 1〜100
- work: 1〜100
- money: 1〜100

スコアに応じてテンプレートを選択する。

### 12.4 テンプレート選択

条件：

- fortune_type
- theme
- score range
- active status
- tags

候補が複数ある場合はseedで1つ選ぶ。

### 12.5 有料本文

有料本文は無料本文よりも具体的にする。

無料：

- 状況の要約
- 前向きな一言
- 軽いアドバイス

有料：

- 恋愛/仕事/金運などの詳細
- 具体的な行動
- 注意点
- 1週間/1ヶ月の見通し
- キャラクターからの追加コメント

## 20. テスト観点

### 20.1 占い機能

- 入力なしでエラーになるか
- 不正な日付でエラーになるか
- 同じ入力で同じ結果が出るか
- 日付が変わると日替わり結果が変わるか
- 無料/有料表示が正しく切り替わるか

### 20.2 手相画像

- jpg/pngがアップロードできるか
- 大きすぎるファイルを拒否できるか
- 同意なしでアップロードできないか
- 削除処理ができるか

### 20.3 課金

- Checkoutへ遷移できるか
- Webhookが正しく処理されるか
- 支払い完了後にPremiumになるか
- 解約時にPremiumが停止するか

### 20.4 セキュリティ

- 他人のreadingを見られないか
- admin以外がadminページに入れないか
- APIに直接アクセスして有料本文を取れないか
- RLSが効いているか

## 24. 開発AIへの重要指示

Claude Code は以下を守ること。
